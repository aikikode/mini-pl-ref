...     """Return a list containing the Fibonacci series up to n."""

>>> # Functions can be defined with variable num of argumets
>>> # 1. Default arg values:
>>> def ask_ok(prompt, retries=4, complaint='Yes or no, please!'):
...     while True:
...         ok = raw_input(prompt)
...         if ok in ('y', 'ye', 'yes'):
...             return True
...         if ok in ('n', 'no', 'nop', 'nope'):
...             return False

...         if retries < 0:
...             raise IOError('refusenik user')

>>> # The default values are evaluated at the point
>>> # of function definition in the defining scope
>>> i = 5

...     print arg

>>> def f(a, L=[]):
...     L.append(a)
...     return L

...    if L is None:

>>> # 2. Functions can also be called using keyword
>>> #    arguments of the form 'kwarg=value':
>>> def parrot(voltage, state='a stiff', action='voom', type='Norwegian'):
...     pass
>>> parrot(1000)                                  # 1 positional arg
>>> parrot(voltage=1000)                          # 1 keyword arg
>>> parrot(voltage=1000000, action='VOOOOOM')     # 2 keyword args
>>> parrot(action='VOOOOOM', voltage=1000000)     # 2 keyword args
>>> parrot('a million', 'bereft of life', 'jump') # 3 positional args
>>> parrot('a thousand', state='pushing up')      # 1 positional, 1 keyword
>>> # THESE CALLS ARE WRONG:
>>> parrot()                     # required arg missing
>>> parrot(voltage=5.0, 'dead')  # non-keyword arg after a keyword arg
>>> parrot(110, voltage=220)     # duplicate value for the same arg
>>> parrot(actor='John Cleese')  # unknown keyword arg

array 
For most purposes List will do. 'array.array' is useful when you
need a homogeneous C array of data for reasons other than doing math.

Arrays are stored more efficiently: as contiguous blocks of memory,
but Lists are faster, because operations on array "raw" data need
to continuously create and destroy python objects when reading from
or writing to the array.

map/dict 
A pair of braces creates an empty dictionary: {}. Placing a comma-separated
list of key:value pairs within the braces adds initial key:value pairs to
the dictionary; this is also the way dictionaries are written on output.

The main operations on a dictionary are storing a value with some key and
extracting the value given the key. It is also possible to delete a
key:value pair with del. If you store using a key that is already in use,
the old value associated with that key is forgotten. It is an error to
extract a value using a non-existent key.
>>> tel = {'jack': 4098, 'sape': 4139}
>>> tel['guido'] = 4127  # add new elem
>>> tel
{'sape': 4139, 'guido': 4127, 'jack': 4098}
>>> tel['jack']
4098
>>> del tel['sape']    # delete the element with the key 'sape'
>>> tel['irv'] = 4127
>>> tel
{'guido': 4127, 'irv': 4127, 'jack': 4098}
>>> tel.keys()         # get a list of all keys
['guido', 'irv', 'jack']
>>> 'guido' in tel
True
>>> # Build dict from tuples of key-value pairs
>>> dict([('sape', 4139), ('guido', 4127), ('jack', 4098)])
{'sape': 4139, 'jack': 4098, 'guido': 4127}
>>> dict([(x, x**2) for x in (2, 4, 6)])  # use a list comprehension
{2: 4, 4: 16, 6: 36}
>>> # ...or simply strings
>>> dict(sape=4139, guido=4127, jack=4098)
{'sape': 4139, 'jack': 4098, 'guido': 4127}

class 
Classes are created at runtime and can be modified further.
All class members are public, all member functions are virtual.
Classes are objects.
Built-in types can be used as base classes.
Class Object is a wrapper around the contents of the namespace created by
the class definition. The class object is bound to the class name.
class TheThing:
    def __init__(self):
        self.number = 0

    def some_function(self):
        print "I got called."

    def add_me_up(self, more):
        self.number += more
        return self.number

a = TheThing()           # class instantiation
TheThing.some_function() # attribute reference
a.some_function()        # also attribute reference
# a.some_function() === TheThing.some_function(a)
